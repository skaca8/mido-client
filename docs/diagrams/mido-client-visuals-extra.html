화면 5. ChannelContext 라이프사이클

ThreadLocal + MDC 가 어떤 시점에 set/clear 되는지

stateDiagram-v2 [*] --> Idle: 새 스레드 Idle --> Bound: withDefaultChannelAction("action")\nset(channel + "." + action) Bound --> Logging: RestClient 호출\nLoggingInterceptor 가\nChannelContext.getChannelAction() 읽음 Logging --> Bound: 다음 인터셉터 흐름 Bound --> Cleared: finally\nChannelContext.clear() Cleared --> Idle: 스레드 풀로 반환 Bound --> Error: 예외 발생 Error --> Cleared: finally 가 여전히 clear() 호출 note right of Cleared ThreadLocal.remove + MDC.remove ThreadLocal 누수 방지 end note note right of Bound MDC 패턴 %X{channelAction} 에서 즉시 사용 가능 end note

finally 블록 덕에 예외 경로에서도 반드시 clear()가 호출됩니다. 다만 비동기로 스레드가 바뀌면 ThreadLocal/MDC는 따라가지 않으므로 별도 전파가 필요합니다.
